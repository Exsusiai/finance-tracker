# Finance Tracker — 产品开发需求文档 (PRD)

> 作者: 项目所有者
> 日期: 2026-05-03
> 版本: v1.0

## 1. 背景与动机

我想要同时实现**记账**和**个人资金管理**两类需求。市面上没有一款现成软件能够同时满足，因此决定自研。

## 2. 核心功能需求

### 2.1 自动记账

#### (a) 手动导入 PDF
- 我可以在每月月底下载各种**银行卡**和**信用卡**的 PDF 账单并上传到软件中
- 软件需要**自动识别**账单内容
- 自动**对账目进行分类**

#### (b) 直接连接银行
- 通过银行提供的接口方案（例如 Open Banking / GoCardless / Nordigen 等）
- 让软件**实时连接银行账户**，获取最新的账单和记录
- 实现**完全自动化的记账**

#### (c) 分类功能
- 内容根据日常支出需求分类，**支持用户自行编辑定义**，**支持多级分类**
- 自动根据账单内容放到相应的列表
- 实现逻辑可以为：
  - **关键字匹配**
  - **调用模型思考**（LLM）
  - 其他启发式
- 如果不确定账单内容，把内容放进**"不确定"列表**等待用户确认
- 支持分类**可编辑**，以及用户**修改分类错误**的支出类别
- **带记忆功能**：如果用户行为把某一个支出放在某一个类别，软件需要**记住**——下次出现类似账单不要放错

### 2.2 资产实时跟踪

我的资产种类较多且分布广泛，包括：
- 人民币 (CNY)
- A 股股票
- 欧洲股票
- 各类**加密货币**及加密钱包资产
  - **链上自动同步**（用户诉求 2026-05-03）：新建"加密钱包"账户时，**只需提供钱包公钥/地址**，软件应自动读取该地址在链上的资产组合（原生币 + ERC-20 / SPL token / UTXO 等），并写入持仓。无需手动逐项添加资产
- 欧元现金
- 黄金

由于这些标的价格波动较大，我很难实时掌握总资产的变化。因此需要软件能够：
- **连接市场接口**获取**汇率**和**资产价格**的实时波动数据
- **实时显示**当前所有资产的**总额**

### 2.3 个人资金 & 现金流记录与分析

需要一个个人储蓄/资金记录表，能够生成随时间变化的图表，涵盖：
- (a) 每月**收入**
- (b) 每月**支出**
- (c) 每月**储蓄**
- (d) 其他**现金流变化**

通过这些直观的图表，可以清晰地看到个人资金的流向和现金流的变化。

## 3. 产品形态

### 3.1 形式
- (a) **前期主要以 Web 页面**的形式呈现，提供一个直观的外部 UI 来展示各项数据
- (b) 后续可以考虑**开发 APP**，但现阶段以 **Web 端优先**

### 3.2 UI 功能模块
- (a) **PDF 上传区域**：支持用户直接上传文件
- (b) **个人资金变化图表**：直观展示总资产的波动情况
- (c) **记账功能模块**：展示每月支出明细及具体分类

## 4. Agent 接口支持

除了面向人类用户的 UI，该软件还必须提供针对 **Agent 的接口**，以便本地 Agent 能够：
- (a) **实时查看和调用**个人资金情况
- (b) 实现**自动化记账**：例如把 PDF 直接发给 Agent，Agent 能调用接口完成记账
- (c) **实时交互**：当我询问最近的资金情况时，Agent 能直观地反馈资金变化
- (d) 也就是说，这个软件在拥有用户 UI 的同时，**还要具备面向本地 Agent 的接口**（MCP / skills / API 等）

## 5. 数据存储

数据需要存储在**两个地方**：

### 5.1 本地数据库
- 可考虑 **SQLite 或 PostgreSQL**，具体选型由实现方决定

### 5.2 Notion 平台
- 同时把账目数据**存储在 Notion 上**
- Notion 上的具体存储形态可以在**稍后阶段再决定**

## 6. 非目标 / 暂不考虑

- 多用户、登录注册（个人单用户工具）
- 移动端 App（P2 之后再说）
- 投资建议、财务规划顾问（只做记录与展示，不做决策）
